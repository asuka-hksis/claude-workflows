# 要員・案件情報収集ワークフロー

## トリガー
ユーザーが「情報収集をお願いします」と伝えたとき。

## 前提
- アカウント: asuka-higuchi@ex.finn.co.jp にログイン
- SESマッチングシート: https://docs.google.com/spreadsheets/d/1lWlLpi1XIRGb27jryoALFvWLzy17bF3aOnAYwLkG0X8/edit?gid=1560731#gid=1560731

## 作業順序
作業a（要員データ収集）→ 作業b（案件データ収集）の順に実行する。

---

## 作業a. 要員データ収集

### 1. 前回終了日時の確認
- SESマッチングシートの「time」タブを開く
- 最後に「要員データ収集」を終了した日時を取得する

### 2. 今回の開始記録
- timeタブの最後の行に新しく以下を記録する
  - 内容: 「要員データ収集」
  - 開始日時: 現在の日時

### 3. 要員情報の収集
以下の2つのソースから、ステップ1で取得した日時以降の要員情報を全て確認する。

#### ソース1: Gmail
- asuka-higuchi@ex.finn.co.jp に届いたメールのうち、前回終了日時以降のもの
- 要員情報が記載されているメールを対象とする

#### ソース2: Notionのパートナー企業一覧
- パートナー企業一覧ページに記載されたシートのうち、要員情報が記載されているもの

### 4. SESマッチングシートの要員シートへの記入
- 要員シートに情報を記載する（本文の欄以外）
- **重複チェック**: 既に登録済みの要員と同一人物でないことを確認してから追加する
- **要員シート該当場所（G列）は必ず入力すること**: メール由来なら「無し」、シート由来なら該当箇所を記入。入力漏れが発生しやすいので注意

### 5. 本文欄への記入
- 本文の欄には**コメントの形**で、メールの本文やシートの中身を貼り付ける
- 1つのメールで複数の要員が紹介されている場合は、該当箇所のみを貼り付ける

### 6. 終了記録
- timeタブに終了日時と経過時間を記入する

## 作業b. 案件データ収集

### 1. 前回終了日時の確認
- SESマッチングシートの「time」タブを開く
- 最後に「案件データ収集」を終了した日時を取得する

### 2. 今回の開始記録
- timeタブの最後の行に新しく以下を記録する
  - 内容: 「案件データ収集」
  - 開始日時: 現在の日時

### 3. 案件情報の収集
以下の2つのソースから、ステップ1で取得した日時以降の案件情報を全て確認する。

#### ソース1: Gmail
- asuka-higuchi@ex.finn.co.jp に届いたメールのうち、前回終了日時以降のもの
- 案件情報が記載されているメールを対象とする

#### ソース2: Notionのパートナー企業一覧
- パートナー企業一覧ページに記載されたシートのうち、案件情報が記載されているもの

### 4. SESマッチングシートの案件シートへの記入
- 案件シートに情報を記載する（本文の欄以外）
- **重複チェック**: 既に登録済みの案件と同一のものでないことを確認してから追加する

#### 案件シートの列構成（左から順）
| 列 | 項目 | 記入ルール |
|---|---|---|
| A | ID | **未記入**（後で自動採番） |
| B | ステータス | 情報収集時は常に「募集中」 |
| C | 案件タイトル | そのまま記入 |
| D | ソース媒体 | 「メール」「シート」「LINE」の3択 |
| E | メールアドレス | ソース媒体が「メール」の場合のみ記入 |
| F | メール受信日時 | ソース媒体が「メール」の場合のみ記入 |
| G | 案件シート該当箇所 | ソース媒体が「シート」の場合のみ記入 |
| H | 年齢制限 | 「40-」(40歳以上)、「-40」(40歳以下)、「30-40」(30〜40歳)のように記載 |
| I | 常駐？ | 「常駐」「フルリモート」「組み合わせ」「不明」の4択。判断できないものは「不明」 |
| J | 最寄り駅 | 駅名のみ記載（「駅」は不要。例: 横浜）。記載がなければ「記載なし」 |
| K | 外国人OK？ | 「外国人NG」等の記載があればNG、それ以外はすべてOK |
| L | 単価 | 「30-40」(30〜40万円)、「-50」(50万円以下)、「40-」(40万円以上)のように記載。単位不要。時給表記の場合は「時給〇〇円」、金額不明の場合は「スキル見合い」等そのまま記載 |
| M | フリーランスOK？ | 「フリーランスNG」等の記載があれば「いいえ」、それ以外はすべて「はい」 |
| N | 言語 | 必要な言語やスキルを簡潔に記入 |
| O | 本文 | **セルに値は入力しない**。セルへのコメントとして該当箇所のみ貼り付け（ステップ5参照） |

#### ソース媒体ごとのEFG列記入ルール
- **メール** → E列（メールアドレス）とF列（メール受信日時）を記入、G列は記入しない
- **シート** → E列・F列は記入しない、G列（案件シート該当箇所）を記入
- **LINE** → E列・F列・G列すべて記入しない

### 5. 本文欄（O列）への記入
- **セルに値は入力しない**。セルに対する「コメント」として貼り付ける
- 収集したメールやスプレッドシートから、該当の案件情報の箇所のみをコピーして貼り付ける
- 1つのメールで複数の案件が紹介されている場合は、該当案件の箇所のみを貼り付け、不要な箇所は含めないこと

### 6. 終了記録
- timeタブに終了日時と経過時間を記入する

## 注意事項
- 途中で指摘を受けた内容は、2度同じ指摘を受けないようにこのワークフローに反映していく

### コメント入力忘れ防止
- データ入力とO列コメント入力は**1行ずつセット**で行う
- 複数行のデータをまとめて入力した後にコメントをまとめて追加する方式は禁止。必ず「1行分のデータ入力 → その行のO列コメント追加 → 次の行」の順序で進める
- 理由: コメントを後回しにすると入力漏れが発生し、後から元メールを探して再入力する手間が大きいため

### コメント位置ずれ防止
- コメントを追加する前に、必ず**Name Box（セル参照ボックス）**にセル番地（例: O17）を入力してEnterを押し、正確なセルに移動してからCmd+Alt+Mでコメントダイアログを開く
- Cmd+Returnでコメント送信後はビューがスクロールしてアクティブセルが変わるため、**視覚的にセルをクリックして次のコメントを追加してはいけない**
- 次のコメントを追加する際も、再度Name Boxで正確なセル番地を指定してから操作する

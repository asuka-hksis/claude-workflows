# 退勤処理ワークフロー

## トリガー
ユーザーが「退勤処理をお願いします」と伝えたとき。

## 手順

### 1. 日付確認
まず **「何日ですか？」** と確認する。

### 2. Google Calendar確認
- アカウント: asuka@fukusaisha.com（u/0）
- URL例: https://calendar.google.com/calendar/u/0/r/day/2026/2/11
- 指定日のイベントを全て取得（夜遅くのイベントも見逃さないようスクロールして確認）
- 前日から日付をまたぐイベントがないかも確認

### 3. 日報スプレッドシート確認
- ファイル: 樋口明日香.日報（Google Sheets）
- URL: https://docs.google.com/spreadsheets/d/1lj51MfoIPLWPW7kmO0Q8yWqEC0iiOILfVgRB35aRQOw/edit?pli=1&gid=2096732206#gid=2096732206
- タブ: 2026/2
- 該当日のセクション（11行単位）を確認

### 4. H列にタスク所要時間を入力（分単位）
- カレンダーのイベントに対応するタスク行のH列に所要時間を入力
- 同じタスク名が複数ある場合は合算する（例: CA業務が3回 → 合計値を1つのセルに）

### 5. 重複イベントの処理
- 同じ時間帯に複数タスクが重なっている場合、重複時間をタスク数で割って各タスクに配分
- 小数点以下は切り捨て

### 6. 対応タスクがないイベントの処理
- カレンダーのイベントに対応するタスク行がない場合、**雑務** に合算して記入
- 雑務行がなければC列に「雑務」を追加

### 7. 日付をまたぐイベントの処理
- 厳密にその日の分だけ計上（例: 23:45-0:30のイベント → 当日分15分、翌日分30分）

### 8. B列に勤務情報を入力
各日のセクション構造（勤務開始ラベルの下に時刻、勤務終了ラベルの下に時刻、等）：

#### 勤務開始時刻
- 日をまたいで勤務していた場合（前日からの継続）は **0:00**
- それ以外はカレンダーの最初のイベント開始時刻

#### 勤務終了時刻
- 日をまたいで勤務していた場合（翌日への継続）は **0:00**
- それ以外はカレンダーの最後のイベント終了時刻

#### 休憩時間
- 計算: （勤務終了 - 勤務開始）- タスク合計時間 = 残り時間
- 残り時間が60分以上 → **1h**
- 残り時間が60分未満 → **?分** の形式（例: 0分、30分）

#### 離席時間
- 計算: （勤務終了 - 勤務開始）- タスク合計時間 - 休憩時間
- 形式: **?h?m**（例: 7h8m、0h0m、14h15m）

### 9. 翌日の欄に翌日の日付を入力
- すべての手順が完了した後、翌日のセクションに翌日の日付を入力する

## スプレッドシート操作のコツ
- セル移動はName Box（左上のセル参照エリア）をクリック → セルアドレス入力 → Return
- 値入力後はTabで確定（Returnだとカーソルが移動して意図しない位置に行く可能性あり）
- 各セルを1つずつName Boxで指定して入力するのが最も確実

# マッチングワークフロー

## トリガー
ユーザーが「マッチングシートに取り組んでください」と伝えたとき。

## 前提
- SESマッチングシート: https://docs.google.com/spreadsheets/d/1lWlLpi1XIRGb27jryoALFvWLzy17bF3aOnAYwLkG0X8/edit
- 要員シート・案件シート・マッチングシートの3つのタブを使用

## 作業手順

### 1. データの確認
- 要員シートからステータスが「募集中」の要員一覧を取得する
- 案件シートからステータスが「募集中」の案件一覧を取得する

### 2. マッチング判定
以下の項目を比較し、条件が合致するペアを見つける：

| マッチング項目 | 要員シート | 案件シート | 判定基準 |
|---|---|---|---|
| 言語/スキル | O列（言語） | N列（言語） | スキルが案件の要求に合致するか |
| 年齢 | H列（年齢） | H列（年齢制限） | 要員の年齢が制限範囲内か |
| 常駐 | I列（常駐可否） | I列（常駐?） | 要員の常駐可否が案件の要件と合うか |
| 最寄り駅 | J列（最寄り駅） | J列（最寄り駅） | 通勤可能な距離か |
| 外国人 | K列（外国人?） | K列（外国人OK?） | 外国人の場合、案件がOKか |
| 単価 | L列（単価） | L列（単価） | 単価の範囲が合うか |
| フリーランス | N列（フリーランス?） | M列（フリーランスOK?） | フリーランスの場合、案件がOKか |
| 経験年数 | M列（経験年数） | ―（案件側に経験年数列がある場合） | 経験年数が十分か |

### 3. マッチングシートへの記入
- マッチングシートの空き行に移動
- **E列**に要員ID（例: Y-1）を入力
- **H列**に案件ID（例: A-1）を入力
- E列のセルにコメントとして、要員シートの本文コメントを貼り付ける
- H列のセルにコメントとして、案件シートの本文コメントを貼り付ける
- ※1行のデータ入力→コメント追加をセットで行い、次の行に移る

### 4. ステータス更新
- マッチングシートに記入した要員のステータスを「募集中」→「マッチング中」に変更（要員シート）
- マッチングシートに記入した案件のステータスを「募集中」→「マッチング中」に変更（案件シート）

### 5. 次のマッチングへ
- ステータス更新後、次のマッチング候補に進む
- 全ての「募集中」の要員・案件について繰り返す

## マッチングシートの列構成
| 列 | 項目 | 入力方法 |
|---|---|---|
| A | マッチングID | 自動 |
| B | GOorNG | 後で判定 |
| C | ステータス | 後で更新 |
| D | コメント | 後で記入 |
| E | 要員ID | **手動入力** |
| F | 要員元メールアドレス | 自動表示 |
| G | 要員詳細 | 自動表示 |
| H | 案件ID | **手動入力** |
| I | 案件元メールアドレス | 自動表示 |
| J | 案件詳細 | 自動表示 |
| K-X | 比較項目（要員-年齢、案件-年齢制限 等） | 自動表示 |

## 注意事項
- 1つの要員が複数の案件にマッチする場合は、それぞれ別の行として記入する
- 1つの案件が複数の要員にマッチする場合も同様
- マッチングの質を重視し、スキルの合致度が最も重要な判断基準とする

# 要員探しワークフロー

## トリガー
ユーザーが「要員を探してください。」と伝えたとき。

## データソース

### 1. Gmail（過去2週間分のメール）
- アカウント: asuka-higuchi@ex.finn.co.jp
- ブラウザでGmailを開いて確認
- 過去2週間に届いたメールから要員情報を収集

### 2. Notion（パートナー企業一覧 > 要員シート）
- ブラウザでNotionを開いて操作
- 「パートナー企業一覧」ページ内の要員シートを確認
- まだSESマッチングシートに記入されていない要員（同一人物が既存行に含まれていないもの）を対象とする

## 出力先
- Notion内の「SESマッチングシート」に行を追加

## 入力項目

| 項目 | 内容 |
|---|---|
| マッチング日 | 処理実行日の日付 |
| 要員名 | 要員の名前 |
| 要員ソース媒体 | どこから見つけたか（Gmail / Notion要員シート） |
| 要員送信元メールアドレス | メール由来の場合、送信元メールアドレスを記入 |
| 要員メール受信日付 | メール由来の場合、メールの受信日を記入 |
| 要員Sheet該当場所 | 要員シート由来の場合、該当箇所（シート名・位置など）を記入 |

## 重複チェック
- SESマッチングシートに既に登録されている要員名と照合し、同一人物が含まれていない場合のみ新規行として追加する

## 追加作業
- 各行の「要員名」をクリックして開いた中に、参照元のメール本文や要員シートの該当部分のコピーを貼り付ける

## 完了報告
- すべての処理が終わったら、完了報告とともに完了時刻を伝える
